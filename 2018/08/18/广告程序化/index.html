《程序化广告实战》笔记

程序化广告的前世今生

1. 常见的广告形式及业态

  • 搜索引擎广告 SEM SEO PPC
  • 展示类广告：文字链接、按钮广告、通栏/横幅广告、“画中画”广告、竖栏/摩天楼广告、焦点图（幻灯片轮播）广告、悬浮广告、对联广告、跨栏广告、背投/弹出广告、插屏/全屏广告、弹出视频、视频贴片广告、视频暂停广告、角标广告、富媒体广告、全页包装广告
  • 原生广告：信息流
  • 社交广告：社会化

2. 结算方式

  • CPT(Cost-per-Time):按时间付费
  • CPM(Cost per Mille):千人成本
  • CPC(Cost-per-Click):点击成本
  • CPA(Cost-per-Action):根据广告最终投放的效果，即回应或激活的数量收费
  • CPD(Cost-Per-Download):按照每次下载收费，下载就付费，不管是否安装

3. 广告行业上下游大体分布

  • 头部广告主：握有大手笔广告预算，十分注重品牌及产品口碑的广告主
  • 中小广告主：比较重视营销的实效，常常会以SEM、公关社交代营为起点来介入整体的营销链条

4. 程序化广告的概念及发展历程

  • 程序化广告的定义：运用技术手段，对整个数字媒体广告投放过程中的各个环节进行信息化，并通过技术手段衔接为一体的一种工具。
  • 程序化广告出现的主要动因：2012年发展元年，产业需求+技术发展推动整个产业升级
  • 发展历程及模式：传统排期广告投放—>AdNetwork（广告联盟）模式—>剩余流量RTB公开竞价—>流量卖方预留库存PDB模式—>优先交易PD—>私有竞价PA—>Trading Desk(TD)
  • 业内主要玩家：流量卖方、广告交易平台、程序化买方、第三方检测、数据方

程序化广告基础

1. 程序化广告4种典型模式

  • 程序化合约(Automated Guaranteed)
  • 优先交易(Preferred deals)
  • 私有竞价(Private Auction)
  • 公开竞价(Open Auction)

1) PD(Preferred Deal)、PA(Private Auction)、OA(Open Auction)均属于实时竞价RTB范畴，仍是程序化广告购买。
2) 由于PDB“保价保量”，对于结算方式广告主同媒体卖方已提前约定好，且广告主同媒体卖方按第三方监测的曝光数据直接结算，PDB技术供应方仅完成不同流量展示不同广告的工作即可，不参与交易环节。
3) 从买方获取流量的交易成本优势角度看，OA > PA > PD > PDB。
4) 一般从获取的流量质量及对流量获取的优先权角度看，OA < PA < PD < PDB。

2. 接口规范

OpenRTB是程序化广告领域关键的技术接口规范，是IAB制定的RTB竞价广告协议的标准。内容主要涉及业务流程及主要的对象模型、数据模型、示例等。

  • 广告请求对象内实体间关系示意图

  • 广告请求对象各数据段（对象）的简单描述表

  • 竞价响应对象内实体间关系示意图

  • 竞价响应对象各数据段（对象）的简单描述

3. 程序化购买/投放的关键特征

1) 流量按优先级管理
2) 交易管理

3) 卖方诉求

  • 卖方不会因增加某种业务模式而减少对另一种（成熟）业务模式的收入；
  • 卖方不会因为某业务模式冲击另一（成熟）业务模式的价格；
  • 卖方不会为了迎合买家而贱卖；
  • 程序化广告售卖剩余流量在很多媒体方目前仍是“增量”收入，占媒体方整体收入的20%～30%，并非传统稳定的收入；
  • 卖方销售：总销售盘量是否会增加
  • 媒体运营：用户日活数据不可公开
  • 媒体产品：用户的体验及隐私数据的保护
  • 大部分视频媒体私有ADX对不同行业都执行不同的底价政策
  • 对于北京、上海来说，稀缺的视频前贴片资源比其他城市的资源底价高出十倍以上（部分视频媒体存在这个现象）。

4) 目标人群投放

  • 用户行为信息保护
  • 广告请求中携带的广告位及用户数据
  • 目标人群投放

5) 流量卖方同买方常见技术对接模式

  • PC/移动Web媒体：JS代码、API
  • 移动App媒体：SDK、API
  • 视频：VAST(Video AD Serving Template)、VPAID(Video Player Ad Interface Definition)

程序化广告中的大数据基础

1. 人的唯一性标识

1) 人唯一性标识的方式

  • 身份证
  • DNA
  • 手机号
  • CookieID
  • QQ号、微信号
  • 设备ID

2) PC端识别技术

  • Cookie
  • CookieMapping

3) 移动端识别技术

  • 追踪个体用户行为，或满足广告监测的需求
  • 个性化推荐及广告
  • 联通数据孤岛，促进数据流动创造数据价值
  • 移动设备ID类型：UDID、IDFA、IMEI、无线WIFI AP
  • 跨屏识别方法与挑战

2. 受众数据及来源

  • 线上数据、线下数据（用户浏览网络页面的行为数据、站内与销售数据、社交数据）
  • 线下VS线上的行为数据特点（线下：成本大、群体性、更真实）
  • 第一方数据：广告主
  • 第二方数据：广告投放方（媒体方、DSP方等）
  • 第三方数据：第三方监测公司

3. 大数据管理平台

数据要想发挥价值，就需要一个集中采集、存储、处理、分析、输出运用的系统平台，即大数据管理平台(Data Management Platform, DMP)。

1) DMP价值意义

  • 消费者洞察
  • 渠道效率分析
  • 数据营销指导

2) 大数据管理处理流程

  • 数据采集
    典型场景安置代码及采集器采集数据；
    广范围地爬取数据；
    通过API接口同各种系统或第三方系统导入数据
  • 数据清洗/传输
    标签化、分类化、加密化
  • 数据分析
  • 数据运用创造收益

监测注意要点

1. 视频广告投放TA浓度KPI注意事项

TA(Target Audience，广告目标受众)，广告投放目标人群及对象。
TA浓度(TA%)的简单理解可以是一次广告投放下来，广告被多少目标人群收看，或者说覆盖了多少目标人群，即广告投放曝光的人数中目标人群的比率。

  • GRP
    Gross Rating Point/Gross Reach Point，总收视率/总收看率，又称毛点评率
    GRP(%) = Reach(目标人群达到率) x Average Frequency(目标人群收看广告的平均次数) x 100
  • CPRP
    Cost Per Reach Point, 每个GRP要多少钱。
    CPRP = Cost(广告花费) / GRP

  • IGRP
    Internet Gross Rating Point
    iGRP(%) = Reach x Average Frequency x 100 = (TA UV / TA Universe) x (TA PV / TA UV) x 100

  • TA Reach
    TA Reach = 目标城市目标人群看到广告人数(TA UV) / 目标城市目标人群总数(TA Universe)

  • TA N+Reach
    N茨及N次以上收看广告的目标人群到达率。
    TA N+Reach = 目标城市目标人群看到“N次及N次以上”广告的人数（N+TAUV）/ 目标城市目标人群总数(TA Universe)
  • TA% (TA浓度)
    广告投放曝光的人数中目标人群所占比例。
    TA%(TA浓度) = TAUV / TA Universe

2. 广告可见性IAB规范

根据MRC(Media Rating Council)的定义：Banner广告需要至少有50%的像素面积在屏幕上展示1秒以上，才能被视为“可见”的广告展示。

广告可见性测量到三个先决条件：

  • Safe Iframe
  • page geometry(页面坐标)
  • browser optimization(浏览器优化引擎)

广告可见性测量的具体方法：

  • 浏览器内广告的相对坐标
  • 屏幕内广告相对坐标
  • 鼠标坐标，屏幕顶点与广告坐标的夹角余弦最大值
  • 浏览器CPU负载计算

影响因素：

  • 广告位在页面中的位置
  • 广告位尺寸
  • 不同行业的内容页也不同
  • 首页（频道首页）及文章内容页也不同

3. 品牌安全

品牌安全(Brand Safety)是近来品牌广告主（尤其是国际大品牌的客户）时常关注的。

  • 推动力动因
    品牌广告主比较在意广告传播的美誉度，不希望广告被展示在与产品服务及品牌形象相悖的媒体环境中。
  • 常见模式
    广告投放后的验证报告、广告投放环节（Bid前，Bid后）
  • 机制及现状
    品牌安全服务主要通过技术手段监测广告曝光时的媒体内容的URL，以及同时抓取页面中的内容全文。
    目前因技术成熟度（是否所有URL都能被分析、分词的准确性、标签的合理性及同广告主的匹配性等问题）、网络环境等问题，导致目前品牌安全的服务还不是很稳定，且大部分只能监测PC上的部分媒体环境。

4. 程序化广告中监测GAP注意事项

  • 地域GAP
    GAP即差距、差异，常常指广告投放观测的数据同第三方监测或广告主观测到数据的差异。
    广告主给媒体结算，一般都是按照第三方监测的数据结果进行的。
    广告主要求的地域定向，往往就是按不同地域、不同的价格及完成量来结算的，一般不会不分地域而按总量来结算，尤其是中视频广告领域。
    广告主严格要求不同地域的曝光量，尤其是视频广告。
    地域GAP的成因：地域GAP是因策略路由导致的（我们常称之为IP漂移）。
  • 结算方式的坑
  • PDB中的损耗问题
  • 移动App端Banner投放的坑
    频次的监测问题、移动App推广归因分析的坑
